# Domain Model

This document describes the **core domain concepts** of the Pharmacy Assistant backend.

The system focuses on promotional catalog items that can be recommended as gifts or suggested by an AI assistant.

The domain is intentionally **simplified** and does not attempt to model a full pharmacy inventory system.

---

# Core Domain Concepts

## Offer

Represents a promotional catalog item that can be recommended by the assistant.

Offers are optimized for **discovery and recommendation**, not full inventory management.

Typical examples:

- perfumes
- wellness kits
- vitamins
- seasonal promotional bundles

### Key Fields

- id
- title
- description
- price
- category
- tags (example: "for_him", "for_her", "birthday", "premium")
- active
- validFrom
- validTo
- createdAt
- updatedAt

---

## FAQ

Represents curated knowledge used to assist responses generated by the assistant.

FAQs provide **reliable grounding information** to avoid incorrect or hallucinated responses from AI models.

Example questions:

- "Do you offer delivery?"
- "What is your return policy?"

### Key Fields

- id
- question
- answer
- category
- active

---

## Inquiry

Represents a **user interaction recorded by the system**.

Each inquiry stores:

- the original user message
- the assistant response
- metadata useful for analytics and debugging

This allows the system to track interactions and improve recommendation strategies.

### Key Fields

- id
- channel (web, whatsapp, api)
- userMessage
- detectedPreferences
- recommendedOfferIds
- assistantResponse
- createdAt

---

## AssistantContext (Non-Persistent)

Represents a temporary object used internally by the assistant service.

This object **is not stored in the database**.

It is used to construct the context sent to the AI provider.

Typical contents:

- user preferences (gender, occasion, budget)
- relevant offers
- relevant FAQs
- system constraints
