# Domain Model

This document describes the core domain concepts used by the Pharmacy Assistant backend.

The system focuses on promotional catalog items that can be recommended as gifts or suggested by an AI assistant.

---

## Core Concepts

### Offer
Represents a promotional catalog item that can be recommended by the assistant.

Offers are optimized for discovery and recommendation rather than full inventory management.

Typical examples include perfumes, wellness kits, vitamins, or seasonal promotional bundles.

Key fields:

- id
- title
- description
- price
- category
- tags (e.g. "for_him", "for_her", "birthday", "premium")
- active
- validFrom
- validTo
- createdAt
- updatedAt

---

### FAQ

Represents curated knowledge used to assist responses generated by the assistant.

FAQs provide reliable grounding information to avoid incorrect or hallucinated responses from AI models.

Examples:

- "Do you offer delivery?"
- "What is your return policy?"

Key fields:

- id
- question
- answer
- category
- active

---

### Inquiry

Represents a user interaction recorded by the system.

Each inquiry stores the original user message, the assistant response, and metadata useful for analytics and debugging.

This allows the system to track interactions and improve recommendation strategies.

Key fields:

- id
- channel (web, whatsapp, api)
- userMessage
- detectedPreferences
- recommendedOfferIds
- assistantResponse
- createdAt

---

### AssistantContext (Non-persistent)

A temporary object used internally by the assistant service to construct the context sent to the AI provider.

This object is not stored in the database.

It contains the relevant information needed for the assistant to generate a response.

Typical contents:

- user preferences (gender, occasion, budget, etc.)
- relevant offers
- relevant FAQs
- constraints
